## Processing raw data
First, ensure you have the additional requirements installed
```bash
pip install -r requirements.txt
```
We all operations are assumed to run in the `scripts/data` folder
```bash
cd scripts/data
```

### Step 1: Preprocess CCD dictionary
A preprocessed version is provided by the Boltz team:
```bash
wget https://boltz1.s3.us-east-2.amazonaws.com/ccd.pkl
```
To do this yourself, use
```
wget https://files.wwpdb.org/pub/pdb/data/monomers/components.cif
python ccd.py --components components.cif --outdir $PWD
```
The next steps are designed to interact with this data via a Redis server. To generate the Redis database, in one process start the database via
```bash
redis-server --dbfilename ccd.rdb --port 7777
```
and in a separate process, run
```bash
python dict_to_rdb.py --dict ccd.pkl
```
After the python script finishes, feel free to stop the redis server.
### Step 2: Generate cluster definitions
Our data processing pipeline requires generating a mapping `clustering.json/clustering.rdb` which maps each structure identifier to a cluster ID.
We do this differently per dataset:
- For the PDB portion of the dataset, we rely on the 40% sequence identity clusters generated by the PDB: this is provided in `data/framediff_boltz`
- For the AFDB portion of the dataset, we rely on the AFDB Foldseek cluster mapping: this is provided in `data/afdb_512_clusters_boltz`

### Step 3: Process structures
First, set up one Redis server for the CCD database and one Redis server for the cluster mappings:
```bash
redis-server --dbfilename ccd.rdb --port 7777
```
```bash
redis-server --dbfilename clustering.rdb --port 7778
```
To featurize a folder of mmCIF structures, use
```bash
python scripts/data/rcsb.py \
    --datadir <folder of mmCIF files (can be nested)> \
    --outdir <parsed data folder> \
    --use-assembly \
    --max-file-size 7000000 \
    --ccd-redis-port 7777 \
    --clusters-redis-port 7778 \
    --num-processes=<num cpus>
```
